# Landing Page - Ouro Car

## 📄 Sobre o Projeto

Este projeto é uma landing page desenvolvida para o portfólio. Inicialmente, este projeto foi concebido e desenvolvido utilizando **WordPress e Elementor** durante minha passagem por uma agência de marketing.

Devido à falta de um backup da versão original desenvolvida no CMS, o projeto foi **replicado manualmente utilizando HTML, CSS e JavaScript puros (Vanilla)**. Por conta dessa transição e da ausência dos arquivos e configurações originais, esta versão replicada pode conter algumas pequenas limitações ou diferenças em relação à versão original desenvolvida no Elementor, mas a essência do design, a responsividade e a estrutura principal foram fielmente mantidas.

## 🛠️ Tecnologias Utilizadas

O projeto foi recriado seguindo as melhores práticas de desenvolvimento front-end com tecnologias nativas:

*   **HTML5:** Estruturação semântica e acessível.
*   **CSS3:** Estilização, animações, layout responsivo (Media Queries, Flexbox/Grid). Nenhuma biblioteca ou framework CSS externo foi utilizado.
*   **JavaScript (ES6+):** Interatividade da página, manipulação do DOM e lógicas de interface, tudo feito com Vanilla JS.

## 🚀 Como Executar o Projeto

Como o projeto é construído apenas com tecnologias web estáticas, não é necessário instalar dependências ou rodar servidores complexos:

1.  Clone este repositório ou baixe os arquivos para a sua máquina.
2.  Navegue até a pasta do projeto.
3.  Dê um duplo clique no arquivo `index.html` para abri-lo diretamente em seu navegador web (Google Chrome, Mozilla Firefox, Safari, Edge, etc.).
4.  *Opcional:* Para uma melhor experiência de desenvolvimento, você pode utilizar a extensão "Live Server" do VS Code ou similar.

## 📁 Estrutura de Arquivos

A organização do repositório segue um padrão simples e direto:

*   `index.html`: Arquivo principal e ponto de entrada da aplicação.
*   `resources/`: Diretório contendo todos os ativos estáticos:
    *   `css/`: Arquivos de folha de estilo.
    *   `js/`: Arquivos de script JavaScript.
    *   *(Outras subpastas para imagens, fontes e ícones podem estar presentes conforme a necessidade do layout).*
